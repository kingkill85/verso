# Internationalization (i18n)

## Summary

Add multi-language support to Verso's web UI using `react-i18next` with automatic browser language detection. 7 languages: English (default), German, French, Spanish, Italian, Portuguese, Dutch.

## Approach

- **Library:** `react-i18next` + `i18next` + `i18next-browser-languagedetector`
- **Detection:** `navigator.language` via browser detector plugin, falls back to English
- **Translation files:** JSON per language in `packages/web/src/locales/{lang}.json`
- **Usage:** `useTranslation()` hook returns `t()` function, replace all hardcoded strings
- **No manual language picker** — purely automatic from browser setting
- **Scope:** Web UI only. Server error messages stay in English.

## i18n setup file

`packages/web/src/i18n.ts` — initializes i18next with browser detector, imports all locale JSONs, configures fallback to English.

## Files to modify

Every component with hardcoded English strings needs `const { t } = useTranslation()` and string replacements. Key files:

- `home.tsx`, `library.tsx`, `account.tsx`
- `sidebar.tsx`, `top-bar.tsx`
- `books/$id.tsx`, `shelves/$id.tsx`, `shelves/new.tsx`
- `login.tsx`, `setup.tsx`
- `admin/users.tsx`
- `confirm-dialog.tsx` (callers pass translated strings)
- `stats.tsx`, `upload.tsx`, `import.tsx`
- `continue-reading-row.tsx`, `add-to-shelf-menu.tsx`
- `auth-form.tsx`, `book-detail overflow menu`

## Non-goals

- Server-side translations (error messages stay English)
- Manual language picker UI
- RTL language support
- Translation of book metadata (titles, authors — those are user data)
- Translation of shelf names (user-created content)
