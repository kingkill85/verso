# GitHub Actions CI/CD Pipeline — Design Spec

## Overview

Two GitHub Actions workflows for Verso: CI checks on PRs/pushes and Docker image releases on version tags.

## Scope

### In scope
1. **CI workflow** — lint, test (with coverage), build on PRs and pushes to main
2. **Release workflow** — build and push Docker image to ghcr.io on version tags

### Out of scope
- Multi-platform Docker builds (linux/arm64) — add later if needed
- Security audits (`pnpm audit`)
- Matrix testing across Node versions
- Deployment automation (deploy-on-release)

---

## 1. CI Workflow (`.github/workflows/ci.yml`)

**Triggers:**
- `pull_request` targeting `main`
- `push` to `main`

**Job: `ci`**
- Runs on: `ubuntu-latest`
- Node: 20

**Steps:**
1. `actions/checkout@v4`
2. `pnpm/action-setup@v4` with `version: 9`
3. `actions/setup-node@v4` with `node-version: 20` and `cache: pnpm`
4. `pnpm install --frozen-lockfile`
5. `pnpm lint`
6. `pnpm test` with coverage flag (vitest `--coverage`)
7. `pnpm build`
8. `actions/upload-artifact@v4` — upload coverage report, retention 7 days

**Notes:**
- pnpm store is cached automatically via `actions/setup-node`'s `cache: pnpm` option
- Coverage is generated by vitest's built-in coverage provider (needs `@vitest/coverage-v8` dependency)
- Build step validates TypeScript compilation succeeds

---

## 2. Release Workflow (`.github/workflows/release.yml`)

**Trigger:**
- `push` of tags matching `v*` (e.g., `v1.0.0`)

**Job: `release`**
- Runs on: `ubuntu-latest`

**Steps:**
1. `actions/checkout@v4`
2. `docker/login-action@v3` — login to ghcr.io using `GITHUB_TOKEN`
3. `docker/metadata-action@v5` — extract tags from git tag:
   - `type=semver,pattern={{version}}` → `1.0.0`
   - `type=semver,pattern={{major}}.{{minor}}` → `1.0`
   - `type=raw,value=latest` → `latest`
4. `docker/build-push-action@v6` — build using repo Dockerfile, push to ghcr.io

**Image:** `ghcr.io/kingkill85/verso:<tag>`

**Usage:**
```bash
git tag v1.0.0
git push origin v1.0.0
# → triggers build + push of ghcr.io/kingkill85/verso:1.0.0
```

---

## 3. Required Changes

### New files
- `.github/workflows/ci.yml`
- `.github/workflows/release.yml`

### Dependency addition
- `@vitest/coverage-v8` as devDependency in `packages/server` (for coverage reporting)

### Package.json script update
- Add `test:coverage` script in `packages/server/package.json`: `vitest run --coverage`
- CI workflow calls `pnpm --filter @verso/server test:coverage` instead of `pnpm test`

---

## 4. Permissions

Both workflows use only the default `GITHUB_TOKEN`:
- CI: read-only (checkout, no writes)
- Release: `packages: write` permission for pushing to ghcr.io

No additional secrets needed.
